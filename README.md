# 导盲系统

## 项目架构

本项目是一个基于多模态AI的导盲系统，整合了YOLO目标检测、VDA深度估计、FunASR语音识别和Qwen大语言模型，为视障人士提供实时的环境感知和导航辅助。

## 系统功能

1. **多模态感知**：
   - 5路摄像头实时采集
   - YOLO目标检测
   - VDA深度估计
   - 语音识别与唤醒
   - 多模态大语言模型

2. **时空融合**：
   - YOLO与VDA帧同步
   - 目标距离计算
   - 目标跟踪与速度计算
   - 环境元数据标准化

3. **智能调度**：
   - 实时安全调度（最高优先级）
   - 复杂场景智能调度
   - 资源管理与异常处理
   - 危险评估引擎（AHP+模糊综合评价）

4. **执行输出**：
   - 语音播报优先级调度
   - TTS文本转语音

5. **模拟调试**：
   - 摄像头模拟
   - 可视化调试界面

## 快速开始

### 安装依赖

**Jetson（推荐）**：先按 [JETSON_PYTORCH.md](JETSON_PYTORCH.md) 安装与 JetPack 匹配的 `torch`、`torchvision`、`torchaudio`，再安装其余依赖：

```bash
pip install -r requirements-jetson.txt
```

**x86 桌面**：先按 [PyTorch 官网](https://pytorch.org) 安装 CUDA 版 `torch` 三件套，再执行：

```bash
pip install -r requirements.txt
```

（`requirements.txt` 不含 `torch*` 固定版本，避免在 Jetson 上误装不兼容的 PyPI 轮子。）

### 配置说明

1. **主配置文件** (`config/config.yaml`)：
   - 系统配置（模拟模式/真实模式）
   - 摄像头配置
   - 模型配置
   - 目标跟踪配置
   - 危险评估配置
   - 资源管理配置
   - 执行输出配置
   - 模拟配置

2. **危险分级规则** (`config/risk_rules.yaml`)：
   - 目标类别权重（车辆、行人、障碍物等）
   - 距离系数（距离越近，风险越高）
   - 速度系数（速度越快，风险越高）
   - 危险分级（level1-4，从高危到安全）
   - 场景复杂度评分（交通信号灯、人行横道、交叉路口、施工区域）

### 启动系统

在vda环境下运行：

```bash
source activate vda && python main.py
```

或直接运行：

```bash
python main.py
```

### 调试模式

在 `config/config.yaml` 中设置 `system.debug: true` 启用可视化调试界面。

## 模块说明

### 1. 多模态感知层

- **摄像头采集**：支持5路摄像头独立采集，每路可单独配置帧率、分辨率。
- **YOLO目标检测**：识别环境中的目标，如行人、车辆、障碍物等。
- **VDA深度估计**：计算目标到摄像头的距离。
- **语音识别**：识别用户的语音指令和唤醒词。
- **大语言模型**：处理复杂场景，提供智能导航建议。

### 2. 目标级时空融合层

- **帧同步**：解决YOLO与VDA的推理时延差异问题。
- **距离计算**：基于深度图计算目标的真实距离。
- **目标跟踪**：跨帧跟踪目标，计算目标速度。
- **元数据封装**：标准化环境感知结果，为调度决策提供输入。

### 3. 核心调度决策层

- **实时安全调度**：评估环境危险等级，生成告警。
- **复杂场景调度**：处理复杂场景，调用大语言模型提供智能建议。
- **资源管理**：监控系统资源，处理异常情况。
- **危险评估引擎**：
  - AHP层次分析法计算各因素权重（距离、速度、类别、场景复杂度）
  - 模糊综合评价对各因素进行评分
  - 特殊场景处理（人行横道、交叉路口、施工区域等）
  - 静止车辆特殊处理（近距离静止车辆视为障碍物）
  - 风险分级与告警触发

### 4. 执行输出层

- **语音播报调度**：根据优先级调度语音播报。
- **TTS引擎**：将文本转换为语音。

### 5. 模拟调试模块

- **摄像头模拟**：模拟5路摄像头输入。
- **可视化调试**：显示摄像头画面、目标框、深度图和危险等级。

## 硬件要求

- **主机调试**：x86主机，8GB以上内存。
- **部署设备**：Jetson Nano 8GB或更高配置。

## 注意事项

1. **模型路径**：确保模型路径正确配置，YOLO模型位于 `perception/yolo/model/yolov8n.pt`，LLM模型位于 `/root/autodl-tmp/qwen35/tclf90/Qwen3___5-4B-AWQ`，ASR模型位于 `/root/autodl-tmp/funasr_models/modelscope_cache/iic/SenseVoiceSmall`。
2. **环境配置**：建议在vda环境下运行，确保所有依赖已正确安装。
3. **显存要求**：LLM模型需要至少6GB显存，建议使用GPU运行。
4. **视频处理**：系统会处理视频文件 `video/video.mp4`，并生成输出视频和结果文件到 `output/` 目录。
5. **帧率设置**：默认每25帧处理一次YOLO和VDA，可在配置文件中调整。

## 扩展建议

1. **传感器扩展**：添加更多传感器支持，如IMU、激光雷达等，提升环境感知能力。
2. **模型优化**：优化模型推理速度，适应边缘设备，如使用模型量化、剪枝等技术。
3. **导航功能**：增加导航路径规划功能，提供更智能的路线建议。
4. **移动端APP**：开发移动端APP，实现远程监控和辅助。
5. **风险评估优化**：
   - 加入动态风险评估，考虑目标运动方向
   - 添加环境因素（天气、光照等）对风险的影响
   - 利用历史数据优化风险评估模型
   - 实现个性化风险阈值调整
6. **多语言支持**：扩展语音识别和TTS支持多语言。
7. **无障碍接口**：增加与其他无障碍设备的接口，如盲杖、智能手表等。
