## 2. 整体数据流与调度逻辑

### 2.1 数据流流程

1. **数据采集**：
   - 摄像头模拟器生成视频帧数据
   - ASR模块接收音频输入

2. **数据处理**：
   - 帧同步模块收集多摄像头数据
   - YOLO模块检测目标
   - VDA模块生成深度图
   - 深度融合模块计算目标距离
   - 目标跟踪模块计算目标速度和轨迹
   - 元数据封装模块整合所有信息

3. **决策与调度**：
   - 实时安全调度评估危险等级
   - 复杂场景调度处理复杂情况
   - 语音播报调度管理告警和提示

4. **执行输出**：
   - TTS引擎将文本转换为语音
   - 调试查看器显示系统状态

### 2.2 调度逻辑

1. **实时安全调度**：
   - 基于目标的距离、速度、类别计算危险分
   - 根据危险分阈值生成不同级别的告警
   - 实现告警冷却和连续告警检测

2. **复杂场景调度**：
   - 当场景复杂度超过阈值时触发
   - 调用LLM分析场景并提供导航建议
   - 管理LLM资源的加载和释放

3. **语音播报调度**：
   - 基于优先级队列管理播报消息
   - 实现告警冷却，避免重复播报
   - 优先处理高优先级告警

## 3. 功能模块详细实现

### 3.1 核心模块

#### 3.1.1 资源管理器 (ResourceManager)
- **功能**：管理系统资源，协调模块间资源分配
- **实现**：通过资源监控和申请机制，确保系统资源合理分配
- **关键方法**：`request_resources()`、`release_resources()`、`update_heartbeat()`

#### 3.1.2 实时安全调度器 (RealtimeScheduler)
- **功能**：评估危险等级，生成实时告警
- **实现**：
  - 计算单个目标的危险分（基于类别权重、距离系数、速度系数）
  - 计算场景复杂度
  - 评估危险等级并生成告警
  - 实现告警冷却和连续告警检测
- **关键方法**：`process_metadata()`、`_calculate_risk_score()`、`_evaluate_risk_level()`

#### 3.1.3 复杂场景调度器 (ComplexSceneScheduler)
- **功能**：处理复杂场景，调用LLM分析
- **实现**：
  - 管理LLM模型的加载和释放
  - 根据唤醒词和环境信息生成prompt
  - 调用LLM分析场景并生成导航建议
- **关键方法**：`process_complex_scene()`、`handle_wake_word()`、`_generate_prompt()`

### 3.2 感知模块

#### 3.2.1 YOLO检测器 (YoloDetector)
- **功能**：目标检测，识别场景中的物体
- **实现**：使用YOLOv8模型进行目标检测
- **关键方法**：`inference()`、`release()`

#### 3.2.2 VDA深度估计器 (VDADepthEstimator)
- **功能**：生成深度图，计算目标距离
- **实现**：使用Video-Depth-Anything模型生成深度图
- **关键方法**：`inference()`、`release()`

#### 3.2.3 ASR语音识别器 (FunASRRecognizer)
- **功能**：语音识别，检测唤醒词
- **实现**：
  - 使用FunASR模型进行语音识别
  - 检测唤醒词（"你好"、"导盲"、"导航"、"小明"、"小明同学"）
- **关键方法**：`inference()`、`release()`

#### 3.2.4 LLM多模态 (QwenMultimodal)
- **功能**：分析复杂场景，提供导航建议
- **实现**：使用Qwen3.5多模态模型分析图像和环境信息
- **关键方法**：`inference()`、`release()`

### 3.3 融合模块

#### 3.3.1 帧同步 (FrameSync)
- **功能**：同步多摄像头帧数据
- **实现**：维护帧缓冲区，确保多传感器数据的时间同步
- **关键方法**：`add_frame()`、`add_yolo_result()`、`add_vda_result()`、`get_sync_data()`

#### 3.3.2 深度融合 (DepthFusion)
- **功能**：计算目标距离
- **实现**：
  - 从深度图中提取目标区域的深度值
  - 过滤异常值，计算目标距离
- **关键方法**：`calculate_target_distances()`、`_calculate_distance()`

#### 3.3.3 目标跟踪 (TargetTracker)
- **功能**：跟踪目标，计算速度
- **实现**：使用跟踪算法跟踪目标，计算目标速度和轨迹
- **关键方法**：`track_targets()`

#### 3.3.4 元数据封装 (MetadataWrapper)
- **功能**：封装环境感知元数据
- **实现**：整合目标信息、深度信息和时间戳，生成统一的元数据格式
- **关键方法**：`wrap_metadata()`

### 3.4 执行模块

#### 3.4.1 语音播报调度器 (BroadcastScheduler)
- **功能**：管理播报队列，控制播报顺序
- **实现**：
  - 使用优先级队列管理播报消息
  - 实现告警冷却，避免重复播报
  - 优先处理高优先级告警
- **关键方法**：`add_message()`、`_scheduler_loop()`

#### 3.4.2 TTS引擎 (TTSEngine)
- **功能**：文本转语音，实现语音播报
- **实现**：使用TTS模型将文本转换为语音
- **关键方法**：`speak()`、`release()`

### 3.5 模拟模块

#### 3.5.1 摄像头模拟器 (CameraSimulator)
- **功能**：模拟摄像头输入
- **实现**：从视频文件读取帧数据，模拟摄像头输入
- **关键方法**：`start()`、`stop()`、`get_all_frames()`

#### 3.5.2 调试查看器 (DebugViewer)
- **功能**：可视化系统状态
- **实现**：显示视频帧、目标检测结果、深度图和危险等级
- **关键方法**：`start()`、`stop()`、`update_frame()`、`show()`

## 4. 配置与部署

### 4.1 配置文件

- **主配置文件**：`config/config.yaml`
  - 系统配置：模式、调试开关、帧率等
  - 摄像头配置：分辨率、帧率等
  - 模型配置：模型路径、类型等
  - 目标跟踪配置：参数设置
  - 危险评估配置：阈值、冷却时间等
  - 资源管理配置：资源阈值
  - 执行输出配置：TTS类型等
  - 模拟配置：视频路径等

- **风险规则文件**：`config/risk_rules.yaml`
  - 类别权重：不同目标的危险权重
  - 距离系数：不同距离范围的系数
  - 速度系数：不同速度范围的系数
  - 复杂度项：不同目标的复杂度分数

### 4.2 部署流程

1. **环境准备**：
   - 安装依赖：`pip install -r requirements.txt`
   - 下载模型文件到指定路径

2. **配置调整**：
   - 修改`config/config.yaml`中的配置参数
   - 根据实际硬件情况调整模型类型（real或mock）

3. **启动系统**：
   - 运行`python main.py`启动系统
   - 系统会自动加载配置并初始化各个模块

4. **调试与监控**：
   - 开启调试模式可查看可视化界面
   - 查看日志文件了解系统运行状态

## 6. 总结

导盲系统采用模块化设计，实现了从感知、融合到决策、执行的完整流程。系统通过多传感器数据融合，实现了对环境的全面感知和危险评估，能够及时发现并提示潜在危险，同时在复杂场景下提供智能导航建议。

系统架构清晰，模块职责明确，数据流顺畅，调度逻辑合理。通过配置文件可以灵活调整系统行为，适应不同的硬件环境和使用场景。

未来可以通过优化目录结构、提升性能、增强功能等方式进一步完善系统，提高系统的可靠性、实时性和智能化水平，为视障人士提供更安全、更智能的导航辅助。
