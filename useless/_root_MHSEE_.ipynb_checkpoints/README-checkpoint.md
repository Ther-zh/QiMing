# 导盲系统

## 项目架构

本项目是一个基于多模态AI的导盲系统，整合了YOLO目标检测、VDA深度估计、FunASR语音识别和Qwen大语言模型，为视障人士提供实时的环境感知和导航辅助。

### 目录结构

```
导盲系统主目录
├── config/                    # 所有可配置参数
│   ├── config.yaml            # 主配置文件
│   └── risk_rules.yaml        # 危险分级、评分规则单独配置
├── core/                      # 核心调度决策引擎
│   ├── __init__.py
│   ├── realtime_scheduler.py  # 实时安全调度引擎
│   ├── complex_scene_scheduler.py # 复杂场景智能调度引擎
│   └── resource_manager.py    # 资源管控与异常处理引擎
├── perception/                # 多模态感知层
│   ├── __init__.py
│   ├── camera_capture.py      # 摄像头采集与调度
│   ├── yolo/                  # YOLO目标检测模块
│   ├── vda/                   # VDA深度估计模块
│   ├── asr/                   # FunASR语音识别模块
│   └── llm/                   # Qwen多模态大模型模块
├── fusion/                    # 目标级时空融合层
│   ├── __init__.py
│   ├── frame_sync.py          # YOLO+VDA帧同步逻辑
│   ├── depth_fusion.py        # 单帧目标距离计算
│   ├── target_tracker.py      # 目标跟踪与速度计算
│   └── metadata_wrapper.py    # 标准化环境感知元数据封装
├── execution/                 # 执行输出层
│   ├── __init__.py
│   ├── broadcast_scheduler.py # 语音播报优先级调度
│   └── tts_engine.py          # TTS封装
├── simulation/                # 硬件输入模拟模块
│   ├── __init__.py
│   ├── camera_simulator.py    # 5路摄像头模拟
│   ├── audio_simulator.py     # 麦克风音频模拟
│   └── debug_viewer.py        # 可视化调试界面
├── utils/                     # 工具函数
│   ├── __init__.py
│   ├── logger.py              # 统一日志配置
│   ├── config_loader.py       # 配置文件加载
│   └── data_formatter.py      # 数据格式化工具
├── main.py                    # 程序入口
├── requirements.txt           # 依赖清单
└── README.md                  # 项目说明
```

## 系统功能

1. **多模态感知**：
   - 5路摄像头实时采集
   - YOLO目标检测
   - VDA深度估计
   - 语音识别与唤醒
   - 多模态大语言模型

2. **时空融合**：
   - YOLO与VDA帧同步
   - 目标距离计算
   - 目标跟踪与速度计算
   - 环境元数据标准化

3. **智能调度**：
   - 实时安全调度（最高优先级）
   - 复杂场景智能调度
   - 资源管理与异常处理

4. **执行输出**：
   - 语音播报优先级调度
   - TTS文本转语音

5. **模拟调试**：
   - 摄像头模拟
   - 可视化调试界面

## 快速开始

### 安装依赖

```bash
pip install -r requirements.txt
```

### 配置说明

1. **主配置文件** (`config/config.yaml`)：
   - 系统配置（模拟模式/真实模式）
   - 摄像头配置
   - 模型配置
   - 目标跟踪配置
   - 危险评估配置
   - 资源管理配置
   - 执行输出配置
   - 模拟配置

2. **危险分级规则** (`config/risk_rules.yaml`)：
   - 目标类别权重
   - 距离系数
   - 速度系数
   - 危险分级
   - 场景复杂度评分

### 启动系统

```bash
python main.py
```

### 调试模式

在 `config/config.yaml` 中设置 `system.debug: true` 启用可视化调试界面。

## 模块说明

### 1. 多模态感知层

- **摄像头采集**：支持5路摄像头独立采集，每路可单独配置帧率、分辨率。
- **YOLO目标检测**：识别环境中的目标，如行人、车辆、障碍物等。
- **VDA深度估计**：计算目标到摄像头的距离。
- **语音识别**：识别用户的语音指令和唤醒词。
- **大语言模型**：处理复杂场景，提供智能导航建议。

### 2. 目标级时空融合层

- **帧同步**：解决YOLO与VDA的推理时延差异问题。
- **距离计算**：基于深度图计算目标的真实距离。
- **目标跟踪**：跨帧跟踪目标，计算目标速度。
- **元数据封装**：标准化环境感知结果，为调度决策提供输入。

### 3. 核心调度决策层

- **实时安全调度**：评估环境危险等级，生成告警。
- **复杂场景调度**：处理复杂场景，调用大语言模型提供智能建议。
- **资源管理**：监控系统资源，处理异常情况。

### 4. 执行输出层

- **语音播报调度**：根据优先级调度语音播报。
- **TTS引擎**：将文本转换为语音。

### 5. 模拟调试模块

- **摄像头模拟**：模拟5路摄像头输入。
- **可视化调试**：显示摄像头画面、目标框、深度图和危险等级。

## 硬件要求

- **主机调试**：x86主机，8GB以上内存。
- **部署设备**：Jetson Nano 8GB或更高配置。

## 注意事项

1. 首次运行时，系统会自动下载所需的模型文件。
2. 在模拟模式下，系统使用Mock模型，无需真实模型文件。
3. 真实模式下，需要确保模型路径正确配置。
4. 系统支持通过配置文件一键切换模拟模式和真实模式。
